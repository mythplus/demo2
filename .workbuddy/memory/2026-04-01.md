# 2026-04-01 工作日志

## 导入记忆功能修复 - 真正写入数据库
- **问题**: 导入功能虽然调用了 `mem0Api.addMemory()`，但默认 `infer=true` 会让 AI 拆分记忆导致内容不一致；且缺少 `user_id` 的记忆会因后端校验失败
- **修复 import-dialog.tsx**:
  - 添加 `infer: false`（原文整条存储，不让 AI 拆分）
  - 添加 `auto_categorize: true`（AI 自动识别分类标签）
  - 新增 `defaultUserId` 状态 + 输入框，用户可为缺少 user_id 的记忆指定默认值
  - 无 user_id 时兜底使用 "default"
  - 预览步骤新增导入说明提示
  - 导入 `User` 图标和 `Input` 组件
- **后端无需改动**: server.py 的 add_memory 接口已支持 infer/auto_categorize 参数

## 修复 defaultUserId 优先级 bug
- **问题**: `user_id: item.user_id || defaultUserId || "default"` — 原有 user_id 存在时 defaultUserId 永远不生效
- **修复**: 改为 `defaultUserId.trim() || item.user_id || "default"` — 填写了默认ID就覆盖所有记忆
- 更新了 placeholder 和提示文案

## 新增批量导入接口 POST /v1/memories/batch
- **后端 server.py**: 新增 `BatchImportItem/Request/ResultItem/Response` 模型 + `batch_import_memories` 接口
  - 接收 items 数组 + default_user_id + infer + auto_categorize
  - 逐条写入 Qdrant，含 AI 自动分类、metadata 补写、变更日志
  - 返回 total/success/failed + 每条结果详情
- **前端 types.ts**: 新增 `BatchImportItem/Request/ResultItem/Response` 类型
- **前端 client.ts**: 新增 `mem0Api.batchImport()` 方法
- **前端 import-dialog.tsx**: `handleImport` 改用 `mem0Api.batchImport()` 一次性提交，替代原来逐条 addMemory 的方式
  - default_user_id 逻辑由后端统一处理

## Mem0 Graph Memory 模块功能分析
- 完成了 mem0.ai Graph Memory 模块的详细功能分析
- 核心功能：在向量记忆基础上增加实体-关系图谱（节点+边）
- 数据结构：实体（name/type/embedding）+ 关系三元组（source→relation→destination）
- 写入需 2-3 次额外 LLM 调用（提取实体、建立关系、冲突检测）
- 支持后端：Neo4j/Kuzu(嵌入式)/Memgraph/Neptune/Apache AGE
- 复刻评估：需 Kuzu + 前端力导向图可视化，7b 模型下会明显增加延迟
