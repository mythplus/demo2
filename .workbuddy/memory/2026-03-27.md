## Bugfix: 记忆状态不一致问题

**问题**: 同一条记忆在主界面显示"活跃"，筛选页面显示"暂停"。

**根因**: 
1. `get_memories` 有两条代码路径：无 user_id 时用 Qdrant 直接查询（能读到最新 metadata），有 user_id 时用 Mem0 SDK `get_all()` 返回的数据（metadata 可能不含我们通过 `set_payload` 写入的 state/categories）
2. `update_memory` 先调 `m.update(text)` 导致 Mem0 重写 Qdrant payload（覆盖之前的 metadata），然后才 `set_payload` 写入新 metadata，存在顺序/覆盖问题
3. `add_memory` 通过 Mem0 `m.add()` 添加时，Mem0 可能不保留自定义 metadata 中的 categories/state 字段

**修复**:
1. `get_memories`: 统一用 `_get_all_memories_raw()` (Qdrant 直接查询) 替代 `m.get_all()`，user_id 筛选改为内存过滤
2. `get_memory_by_id`: 优先 Qdrant `retrieve` 直接读取，fallback 到 Mem0 SDK
3. `update_memory`: 加日志确认写入，逻辑注释更清晰（先 m.update 文本，再读最新 payload 后写 metadata）
4. `add_memory`: `m.add()` 返回后，补写 `set_payload` 确保 categories/state 落盘到 Qdrant
